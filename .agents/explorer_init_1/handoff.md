## 2. Logic Chain
1. **News Sentiment Mismatch**: The unit and E2E tests expect `get_sentiment` to return a `float`. However, the implementation returns `{"score": score, ...}` which is a dict. Asserting `dict == float` raises failures.
2. **Politician Tracker Mismatch**: The tests assert `data["ticker"] == "AAPL"`, etc. The copy mode returns keys `"composite_score"`, `"trade_count"`, etc. Referencing missing keys raises `KeyError`.
3. **Demo Mode Trigger**: The E2E tests mock API endpoints, but do not set a dummy `ALPACA_API_KEY` env var. The executor checks `_is_configured` which returns `False` because the key is empty, triggering the demo mode (prefix `"demo-"`) instead of contacting the mock server. The assertion `startswith("ord-")` fails.
4. **Macro Context Omission**: `macro_context` is configured as a weight in `config.yaml` but has no implementation, database storage support, or LLM prompt integration.
5. **Syntax Error in test_tier2_boundary.py**: On line 258, `{"indicators": {"vwap": 1.0} * 1000}` tries to multiply a dict by 1000, raising a Python `TypeError` and causing test execution to fail.
6. **Unused Dependencies**: Since `beautifulsoup4`, `aiohttp`, `apscheduler`, and `websockets` have zero occurrences in the import declarations of our python source codebase, they are safe to remove from `requirements.txt`.
